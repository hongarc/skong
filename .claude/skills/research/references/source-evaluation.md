# Source Evaluation Guide

## Reliability Tiers
1. **Official docs** — Most reliable for API/usage info
2. **GitHub repos** — Check stars, last commit, open issues
3. **Stack Overflow** — Check date, votes, accepted answer
4. **Blog posts** — Check author credibility, date published
5. **AI-generated content** — Cross-verify everything

## npm Package Evaluation
- Weekly downloads > 10k = established
- Last publish < 6 months = maintained
- Open issues < 50 = manageable
- TypeScript types included = bonus
- Check license compatibility

## Red Flags
- No updates in 12+ months
- Many unresolved security advisories
- Single maintainer with no activity
- Breaking changes in recent versions without migration guide
- Docs don't match latest version
