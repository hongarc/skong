# Research Checklist for Proposals

Before drafting a proposal, gather:

## Codebase
- [ ] Read relevant existing code
- [ ] Check for similar patterns already in use
- [ ] Identify files that will be affected
- [ ] Check recent git history for related changes

## External
- [ ] npm packages that could help (check downloads, maintenance)
- [ ] Known solutions to similar problems
- [ ] Security implications (OWASP top 10)

## Team Context
- [ ] Who else is working in this area?
- [ ] Any related Jira tickets?
- [ ] AWS team involvement needed?
- [ ] Timeline constraints?

## Technical Constraints
- [ ] Node.js compatibility requirements
- [ ] Deployment target (local/Vercel/AWS)
- [ ] Performance requirements
- [ ] Data migration needed?
