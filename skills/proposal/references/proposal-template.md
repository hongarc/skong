# Proposal Template

```markdown
# [Title]: Brief Problem Statement

## Status
Draft | Under Review | Approved | Rejected

## Context
- What is the problem?
- Why does it need solving now?
- Link to Jira ticket: [TICKET-XXX]

## Current State
- How does it work today?
- What's broken or missing?

## Proposed Solution
### Approach
- Clear description of what we'll build/change
- Key technical decisions

### Scope
- What's included
- What's explicitly NOT included

### Implementation Steps
1. Step with effort size (S/M/L)
2. ...

## Alternatives Considered
| Approach | Pros | Cons |
|----------|------|------|
| Option A | ... | ... |
| Option B | ... | ... |

## Trade-offs & Risks
- Risk 1 → Mitigation
- Risk 2 → Mitigation

## Recommendation
Clear statement: "We recommend [X] because [Y]"

## Dependencies
- Team/service dependencies
- AWS team involvement needed? (Y/N)

## Open Questions
- Questions needing lead/team input
```
