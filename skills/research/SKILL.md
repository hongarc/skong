# Research Skill

## When to Use
- Evaluating libraries or tools
- Understanding a new technology
- Comparing approaches for a problem
- Investigating a bug or issue
- Gathering info for a proposal

## Workflow

### Step 1: Scope
- Define: what exactly are we researching?
- Key terms and constraints
- What decision will this inform?
- Recency requirements (latest Node.js? LTS only?)

### Step 2: Gather
- Search web for current info (max 5 parallel searches)
- Check npm for packages (downloads, last update, issues)
- Read official docs
- Check codebase for existing solutions
- See `references/source-evaluation.md`

### Step 3: Analyze
- Compare findings against requirements
- Identify trade-offs
- Cross-verify claims from multiple sources
- Flag outdated or conflicting info

### Step 4: Report
- Use format in `references/report-format.md`
- Max 150 lines for concise report
- Lead with recommendation
- Include sources

## Key Principles
- Multiple source verification
- Be honest — if something is unclear, say so
- Straight to the point, no filler
- Distinguish facts from opinions
- Always note recency of sources
